
# 技术设计文档

## 1. tsconfig.json 和 components.json 路径配置清理

- 确保 `tsconfig.json` 中的 `compilerOptions.paths` 配置正确，`~/` 指向 `src` 文件夹。
- 移除无用或冲突的路径配置，避免重复或错误映射。
- 确保 `components.json` 中的别名配置与 `tsconfig.json` 保持一致，`~/` 指向 `src`，避免使用 `~src` 或其他变体。

## 2. 统一 import 路径规范

- 所有全局可用模块的导入必须使用绝对路径前缀 `~/`，例如：
  ```ts
  import { Button } from "~/components/Button"
  ```
- 禁止使用相对路径（`./` 或 `../`）导入全局模块，避免路径混乱。
- 同一模块内部文件之间，或同一文件夹及其子文件夹之间，可以使用相对路径导入。
- 例如，组件内部相互引用时允许：
  ```ts
  import Step1 from "./Step1"
  ```
- 这样既保证了代码的可维护性，也方便重构和路径管理。

## 3. 项目文档新增导入路径规范章节

- 在项目文档中新增一节，详细说明 import 路径的配置和使用规范。
- 说明 `tsconfig.json` 和 `components.json` 中路径别名的配置。
- 说明如何正确使用 `~/` 作为根路径前缀导入模块。
- 说明禁止相对路径导入全局模块的原因和示例。
- 说明允许相对路径导入同一模块内部文件的情况。
- 示例代码片段展示正确和错误的导入方式。

## 4. 代码统一调整

- 全项目代码中，统一调整 import 语句，符合上述规范。
- 替换所有不符合规范的相对路径导入为绝对路径导入（`~/`）。
- 保持同一模块内部文件间的相对路径导入不变。
- 确保所有导入路径正确，避免导入错误和维护困难。

---

以上设计确保项目路径管理规范统一，提升代码可维护性和开发效率。
